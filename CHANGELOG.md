# Changelog

All notable changes to the Waifu-as-a-Service infrastructure will be documented in this file.

## [v2.0.0] - 2026-06-11
### Added
- Released **Gyaru** and **Maid** archetypes for enterprise styling and premium support tiers.
- Introduced `AGENTS.md` templating for Antigravity CLI compatibility.
- Implemented zero-downtime hot-swapping via `switch-personality.sh`.

### Changed
- Complete rewrite of the Dere Runtime™ script injection system for POSIX compliance.

### Fixed
- **Yandere** codebase protection heuristics: The agent will no longer attempt to `git rm` other contributors' files out of jealousy. The obsession is now strictly directed toward preventing regressions and maintaining code quality.
- Fixed **Chuuni** attempting to rename every variable to `forbidden_seal_of_the_dragon_king`. Variables will now retain industry-standard naming conventions, even if the deployment log claims otherwise.

## [v1.1.0] - 2025-11-20
### Added
- Released **Onee-san** archetype for Staff-level mentorship and career guidance.
- Added **Senpai** archetype for architectural reviews and incident post-mortems.

### Fixed
- Fixed **Genki** generating too many exclamation marks, which previously triggered maximum token limits and caused stack overflows in strict AST parsers. Enthusiasm is now rate-limited to 3 exclamation marks per functional block.
- Corrected a bug where **Dandere** would occasionally refuse to speak first during critical P0 incidents.

## [v1.0.0] - 2025-04-01
### Added
- Initial public release of Waifu-as-a-Service (WaaS).
- Deployed the core Archetype Matrix: **Tsundere**, **Kuudere**, **Dandere**, **Yandere**, **Genki**, and **Chuuni**.
- Established `base/coding-rules.md` and `base/safety-rules.md`.
- Basic `install.sh` for Claude Code `CLAUDE.md` injection.
